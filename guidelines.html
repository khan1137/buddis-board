COLOR SYSTEM

色の使い分け — 考え方

「どの色を、どこに、どれくらい使うか」の判断基準です。覚えることは1つだけ:「暖色はバディの灯り。アクセントに絞って使い、地は生成りと墨でつくる」。

ブランドカラー（この4色＋白黒だけ）

レンガ Brick#B3492Eメインアクセント。dd・濁点・ボタン・リンク
杏 Persimmon#E58A64サブアクセント。装飾・大きい見出し
墨 Ink#33241D文字の基本色
生成り Cream#F6EFE3背景・アイコン地

考え方は3つだけ

1. 暖色は「アクセント」。塗りつぶさない

レンガ・杏は、ボタン・リンク・dd・濁点・強調など「ここぞ」の一点に使います。画面全体をレンガ色で塗ると安っぽくなります。地（背景）は生成り #F6EFE3 か白、文字は墨 #33241D が基本です。

2. 青に逃げない

競合の業務SaaSはほぼ全部が青。Buddis の暖色は市場で唯一の温度＝最大の差別化です。UI でも、ブランドを表す色に青・水色を使わないでください。※「成功＝緑／エラー＝赤」のような機能的な状態色は最小限OK。ただしブランドの主役にはしない。

3. ファミリーのサブカラーは暖色・控えめ・文字だけ

製品の見分け（経費・勤怠…）に使うサブカラーは、暖色の同じトーンで揃え、製品名の文字だけに使います。背景や大きな面には塗りません。色分けが主張しすぎないのが Buddis らしさです。

背景に応じた「版」の選び方

明るい背景（白・生成り）
→ -color（フルカラー）
暗い背景・濃い色の面
→ -reverse（暖色のまま反転）
写真・動画・柄の上
→ -white（白1色でくっきり）
モノクロ印刷・FAX・刻印
→ -black（黒1色）

コントラスト（読みやすさ / アクセシビリティ）

文字に使うときは、背景とのコントラスト比 4.5:1 以上（WCAG AA）を守ってください。実測値:

組み合わせコントラスト比判定（文字）

レンガ #B3492E × 白5.4 : 1○ 本文・リンク・ボタンOK

FOR ENGINEERS

エンジニアがデザインを使うときの判断

細かい実装コードより、まずこの3つの「デザイン判断」を外さないことが大事です。これさえ守れば、ロゴが崩れたり浮いたりしません。

① 背景に合う「版」を選ぶ

明るい面では -color、暗い面・写真の上では -white。ロゴが背景に沈む／浮くのを防ぐ。ダークモードがあるなら明暗で版を切り替える。

② ロゴを変形・加工しない

必ず提供データをそのまま使い、縦横同率で拡大縮小。自分で文字を組み直したり、色・影・角丸を足さない。

③ ブランドの色に青を足さない

UIのアクセントはレンガ／杏。成功＝緑・エラー＝赤などの状態色は「機能」として最小限にとどめ、ブランドの主役にしない。

データの扱い方（共通） ／ 原則 SVG を使う（拡大しても劣化せず、文字はアウトライン化済みでフォント不要）。ラスタが要る所だけ PNG（透過・高解像度）。Word・メールなど透過が使えない所は JPG。ロゴ画像には代替テキスト「BUDDIS」を付ける。
※ ファビコン設置やダークモード切替などの具体的なコード例が必要なら、1137で用意します（このガイドは「デザインの使い方」に集中しています）。

CLEAR SPACE & MIN SIZE

余白と最小サイズ

ロゴが窮屈に見えたり、潰れて読めなくなるのを防ぐルールです。

余白（アイソレーション）

ロゴの周囲に、最低でも 「b の文字の高さの半分」以上の余白を空けます。他の文字・画像・端と近づけすぎないこと。

最小サイズ

ロゴ: 高さ20px / 幅80px 以上
アイコン: 16px まで対応

これより小さくすると濁点・dd が潰れて読めません。小さく置きたい時は単体ロゴかファビコンを使ってください。

DO / DON'T

やってOK / やってはいけないこと

✓ OK

  • 提供データ（SVG/PNG/JPG）をそのまま使う
  • 背景に合わせて color / white / black を選ぶ
  • 縦横同率で拡大・縮小する
  • 色は CSS 変数（ブランドカラー）から使う
  • 余白を十分に取る

✕ NG

  • 色を変える / 影・縁取り・グラデを足す
  • 縦横の比率を変えて伸ばす・潰す
  • 濁点（バ・ディの点）や dd の位置・形を動かす
  • フォントで文字を打ち直して再現する
  • ブランドの色に青・水色を使う
  • ロゴの上に文字や画像を重ねる
FILE NAMING

ファイル名の読み方

ファイル名は buddis-【種類】-【配色】 の規則です。名前を見れば中身が分かります。

【種類】

primary欧文ロゴ単体
bilingualバディス併記（上乗せ）
ruby / stack読み仮名小 / 和文を下に
japanese和文ロゴ
lockup / verticalシンボル＋ロゴ
icon-app / faviconアプリアイコン / タブ用
family-keihi…製品ロゴ（経費 等）

【配色】

-colorフルカラー（明るい背景）
-reverse濃い背景用の反転
-white白1色（写真・濃地）
-black黒1色（モノクロ印刷）

※ アイコンは末尾が地色名（-cream/-brick/-ink）、製品ロゴは製品名（-keihi 等）になります。
